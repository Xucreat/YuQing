## 二、修改内容

### 2.1 后端 `GET /api/alerts/records`

在既有 `page / size / risk_level / handled` 基础上新增：

```python
status: str | None = None,            # 按处置状态精确筛选
exclude_status: str | None = None,   # 排除指定状态（如 false_positive）
```

- **合法性校验**：`status` / `exclude_status` 取值必须在白名单
  `{pending, processing, resolved, ignored, false_positive}`，非法值返回 **422**（detail 提示允许值）。
- **查询条件**：
  - `if status: q = q.where(AlertRecord.status == status)`
  - `if exclude_status: q = q.where(AlertRecord.status != exclude_status)`
- **默认值均保持 `None`** ⇒ 不影响任何既有无参调用，不影响其它接口（`/unread`、`/rules`、`/evaluate`、处置接口均未改动）。
- 新增模块级常量 `_ALLOWED_ALERT_STATUSES` 与白名单校验，避免重复魔法字符串。

### 2.2 前端 `Alerts.vue`

1. **处置状态筛选下拉**（新增）
   - `recFilterStatus`：el-select（clearable），选项「全部 / 待处理 / 处理中 / 已解决 / 已忽略 / 误报」，对应参数 `status`。
2. **隐藏误报开关**（新增）
   - `hideFalsePositive`：el-switch，**默认 `true`（开启）**。
   - 开启时 `loadRecords()` 发送 `exclude_status=false_positive`；关闭时不发送 ⇒ 显示全部历史。
   - 默认进入预警记录页即隐藏误报。
3. **保留能力**（未删）
   - 风险等级筛选 `recFilterRisk` → `risk_level` ✅
   - 处理状态筛选 `recFilterHandled` → `handled` ✅（保留，未用 status 替代；按交互要求未删除）
   - 分页、查看详情、处置操作 ✅

`loadRecords()` 新增两行：
```ts
if (recFilterStatus.value) params.status = recFilterStatus.value
if (hideFalsePositive.value) params.exclude_status = 'false_positive'
```
