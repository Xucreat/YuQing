## 2. 原型层（apple-style）现状

位置：`prototype/apple-style/`（`index.html` + `app.js`(58KB) + `style.css` + `assets/`）

- **6 个页面全部设计完成**：登录、驾驶舱、舆情列表、舆情详情、事件中心、**预警中心**、**传播溯源**。
- 视觉为统一 apple 风格（圆角卡片、浅色留白、SVG 折线/柱状/网络图），含 toast、弹窗、分页器等交互组件。
- **关键性质**：纯静态 mock 数据（`renderAlerts` / `renderPropagation` 注释明确标注「后端尚未实现，原型为规划演示 / 规划模块」）。**不能当作接口契约直接使用**，仅作视觉与交互参考。
- 比当前实现多出 **预警中心** 与 **传播溯源** 两个页面，是「完善」工作的主要增量来源。

## 6. 后续「完善」工作清单（建议优先级）

**P0 — 视觉对标（你当前在进行的工作）**

- 抽取 apple-style 设计 token（圆角/间距/配色/阴影）与基础组件（Card、Nav、Table、Pill、Chart 容器）。
- 对 5 个现有视图统一换肤，使其接近原型观感；建议用自定义组件覆盖 Element Plus 默认样式，而非全量替换。

**P1 — 预警中心（新模块，前后端均缺）**

- 后端：补充数据模型/派生逻辑（预警可由风险分阈值 + 事件聚合派生，未必需要新表）+ `GET /api/alerts`（列表/筛选/分级统计）。
- 前端：新增 `Alerts.vue` + 路由 `#/alerts` + 左侧导航项 + TS 类型；对齐原型的统计卡（今日预警/红色预警/处理中/已处置）+ 表格 + 级别/状态筛选。

**P2 — 传播溯源（新模块，前后端均缺）**

- 后端：按事件聚合传播链路（起源 → 平台扩散 → 关键节点）+ 时间轴，`GET /api/events/{id}/propagation`。
- 前端：新增 `Propagation.vue` + 路由 + 导航项；复用原型 SVG 网络图思路（或改 ECharts graph），实现节点/链路/时间轴三块。

**P3 — 前端细节缺口**

- 事件详情页 `/event/:id`（关联舆情下钻）。
- 采集触发入口（对接已有 `/api/collector/run`）。
- 统一响应拦截器（401 → 登出跳登录）。

**P4 — 后端清理与加固**

- 删除 `ai_service.py` / `event_service.py` 废弃文件。
- 按需要补 CORS 中间件（生产由 nginx 反代，dev 由 vite 代理，优先级低）。
- 采集状态若需持久化，从内存移入库表。

**P5 — 联调与验证**

- `docker-compose up` 跑通三服务；确认 8080 登录页 → admin/admin123 → Dashboard 有数据。
- 后端 `pytest`（预期 53 passed）；前端 `npm run build` 验证可编译。
- 确认 MockCollector 生成 ≥30 条、事件 ≥3 个、DeepSeek 失败可降级（验收清单）。


## 8. 一句话总结

> 后端骨架与核心链路已经很扎实（53 测试通过、数据库跑过），前端 5 个核心页也已打通；真正的「完善」增量集中在三件事：**(1) 把现有前端视觉对标 apple 原型、(2) 补齐原型里预警中心/传播溯源两个规划模块的前后两端、(3) 跑一次 docker-compose 联调验收**。README 里「Phase 1–4 待执行」的描述已严重过时，实际进度远快于文档记录。
