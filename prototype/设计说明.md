# 大厂县公安互联网舆情监测研判平台 · MVP 前端原型设计说明

> 原型形态：高保真 HTML 交互原型（`index.html`，可点击切换页面）+ 各核心页面 PNG 截图（`png/`）。
> 设计目标：在 MVP 阶段跑通「采集 → 入库 → AI 分析 → 风险判断 → 事件聚合 → 前端展示」闭环，界面专业、克制、可演示。
> 技术约束对齐实现：Vue3 + Element Plus + ECharts + Pinia（原型以静态 HTML/CSS 还原其视觉语言）。

---

## 一、设计原则

1. **政务严肃感 + 数据可读性**：深海军蓝侧边栏（`#1f2d3d`）锚定"公安/政务"属性，内容区浅灰底（`#f0f2f5`）+ 白卡保证信息密度与可读性。
2. **状态即信息**：风险评分、情感、分析状态、风险等级全部用颜色+标签可视化，减少认知负担。
3. **闭环可见**：在驾驶舱顶部常驻"六步流程条"，让使用者随时感知系统当前所处阶段。
4. **组件复用**：统计卡、筛选条、表格、分页、标签为统一组件，跨页面一致，降低设计债。

## 三、信息架构与核心流程映射

```
采集 → 入库 → AI 分析 → 风险判断 → 事件聚合 → 前端展示
  │         │         │           │           │           │
Collector  PostgreSQL AIService  risk_score  EventAgg.   前端页面
POST /api/collector/run
          POST /api/analyze/{id}
                              POST /api/events/aggregate
GET /api/dashboard/stats · GET /api/opinions · GET /api/opinions/{id} · GET /api/events
```

| 流程阶段 | 对应前端页面 | 关键交互 |
| --- | --- | --- |
| 展示（总览）| 驾驶舱 Dashboard | 4 张统计卡 + 7 日趋势 + TOP10 关键词 + 风险分布 |
| 展示（列表）| 舆情列表 Opinions | 来源/情感/关键词筛选 + 分页 + 状态标签 |
| 展示（详情）| 舆情详情 OpinionDetail | 原文 + AI 摘要/情感/风险分/关键词/研判建议 + 触发分析 |
| 展示（聚合）| 事件中心 Events | 手动聚合 + 风险等级 + 关联舆情数 |
| 流程总览 | 核心流程 Flow | 六步闭环 + 接口映射卡片 |

## 五、交互与可用性约定

- **导航**：左侧菜单高亮当前页；详情页高亮"舆情列表"。
- **色彩语义**：红=高/负面/失败，橙=中/分析中，绿=低/正面/完成，蓝=主题与链接。
- **加载/空态**：实现以 `v-loading` 与 `el-empty` 兜底（原型中以静态示意）。
- **响应式**：MVP 以桌面 1440×900 为主；侧边栏固定 220px，内容区最小宽度 1180px（实现中由 Element Plus 栅格自适应）。

## 六、与实现的对应关系（确保"根据当前实现"）

- 侧边栏菜单、配色、Logo 文字均取自 `AppLayout.vue`。
- 驾驶舱四卡（总舆情/今日新增/高风险/事件数）取自 `Dashboard.vue` + `GET /api/dashboard/stats`。
- 风险/情感/状态标签逻辑取自 `Opinions.vue`、`OpinionDetail.vue`、`Events.vue` 中的 `riskColor/sentimentTag/statusTag`。
- 详情页双栏结构、AI 分析字段（摘要/情感/风险分/关键词/研判建议/分析时间/触发分析）取自 `OpinionDetail.vue` + `POST /api/analyze/{id}`。
- 事件中心"手动聚合"按钮与结果展示取自 `Events.vue` + `POST /api/events/aggregate`。
- 核心流程页依据 `README.md` 与 `architecture.md` 的闭环描述绘制。
