## 二、功能①：实时快讯 / 预警滚动 — 自动实时滚动

### 2.1 实现要点（`frontend/src/components/command-screen/FeedList.vue` 重写）
- **无缝循环**：内容渲染为 **双份副本**（`v-for="copy in 2"`），轨道 `translateY(0) → translateY(-50%)` 关键帧动画，到一份高度处恰好接上第二份，视觉无缝。
- **智能滚动（仅溢出时）**：`measure()` 通过 `ResizeObserver` 测量「单份内容高度」与视窗高度，仅当 `单份高度 > 视窗 + 1px` 才加 `.scrolling` 类触发动画；内容不足一屏时**静态展示**，不空转。
- **速度恒定**：`duration = max(条数 × 2.4s, 8s)`，条数越多滚得越慢，单条也平滑。
- **悬停暂停**：`.cs-feed-viewport:hover .cs-feed-track.scrolling { animation-play-state: paused }`，方便阅读。
- **无障碍**：`@media (prefers-reduced-motion: reduce)` 下关闭动画。
- **数据刷新重测**：`watch(rows, ...)` 在条数变化时重新 `measure()`，避免新增/减少后滚动状态失真。
- **零后端改动**：纯表现层，复用既有 `recent` / `alerts` props 与既有预警处置摘要（待处置/已处置固定顶部不随滚动）。

### 2.2 验证结论（puppeteer 真实浏览器）
- 两个 FeedList 轨道均 `copies:2`、`animationName:"cs-feed-scroll-*"、`animPlaying:"running"`、`hasScrollingClass:true` → **自动滚动生效** ✓
- 显示层无裁切（视口 `sw=cw=1440, sh=ch=900`）✓

## 三、功能②：地图点击下钻（真·下钻）

### 3.1 后端新增接口 — `GET /api/dashboard/region-children`

**改动文件**
- `backend/app/schemas/dashboard.py`：新增 `RegionChildCity` / `RegionChildRaw` / `RegionChildrenResponse`。
- `backend/app/services/dashboard_service.py`：新增 `get_region_children(db, province_name, days=7)` 与 `_province_code` / `_city_code` 行政区划代码前缀推导辅助函数。
- `backend/app/api/dashboard.py`：新增路由 `GET /region-children`（`Query` 参数 `province`、`days`，需登录鉴权）。

**关键数据修正（重要）**
初版用 `Region.parent_code` 链式回溯，导致 **大厂回族自治县（`parent_code=None`）被漏计**（河北仅 112 条）。
改为 **GB/T 2260 代码前缀推导**：省 = `code[:2]+"0000"`、市 = `code[:4]+"00"`，与 `parent_code` 字段解耦，县自动上卷到所属市。修正后 **河北省合计 288 条**（廊坊市 196 = 大厂 176 + 香河 10 + 廊坊市 10）。

> 注：接口返回 404 表示「库中无该省细分数据」（如北京等）。前端对此**优雅降级**而非报错，详见 3.3。

### 3.2 前端下钻重构 — `frontend/src/components/command-screen/ChinaMap.vue` 重写

- **国家级**：本地静态资源 `/geo/china-provinces.json`（DataV 同源，不依赖外网），建立 `省名 → adcode` 映射。
- **下钻流程** `drillTo(name)`：
  1. 由映射取省 `adcode` → `loadProvinceGeo(adcode)`（**先本地 `/geo/{adcode}_full.json`，缺失则按需联网 DataV** 并缓存）；
  2. `registerMapOnce('prov-{adcode}', geo)`；
  3. `fetchRegionChildren(name, days)` 取市级上卷数据（404 则降级为「该省暂无细分数据」）；
  4. `renderProvince()` 市级着色 + `trigger:'item'` tooltip。
- **返回**：`backToCountry()` 复位国家级；顶部 `← 返回全国` 按钮（仅省级显示）。
- **交互边界**：`chart.on('click', ...)` 仅在国家级触发，避免下钻态内误触；`days` 变化且处于下钻态时自动重拉细分数据。
- **容错**：GeoJSON 加载/注册失败 → 回退国家级，避免卡在半下钻。

**配套改动**
- `frontend/src/types/index.ts`：新增 `RegionChildCity` / `RegionChildRaw` / `RegionChildren`。
- `frontend/src/composables/useCommandScreen.ts`：新增 `fetchRegionChildren(province, days=7)`。
- `frontend/src/views/CommandScreen.vue`：`<ChinaMap>` 现传入 `:regions` 与 `:days`。
- **GeoJSON 资产**：`frontend/public/geo/130000_full.json`（河北省 11 地级市边界，DataV `130000_full.json`，73KB）经 `_d.py` 部署至 `backend/app/static/geo/130000_full.json`（实测 HTTP 200）。其他省份点击时按需从 DataV 联网拉取并前端缓存。

### 3.3 验证结论（真实浏览器 + 真实 API）
- **河北下钻**：点击河北省 → 市级地图渲染 → 顶部徽标 `已下钻 · 河北省 · 共 288 条` → `← 返回全国` 可见 → 点击返回国家级成功（`backOk:true`）✓
- **悬浮数据**：`tooltip.trigger:'item'`，悬浮显示 `省/市名（含所辖县）<br/>舆情数量：N` ✓
- **无数据省份优雅降级**：北京等返回 404 → 前端展示「已下钻 · 北京市 · 该省暂无细分数据」，不报错 ✓
- **显示层**：下钻态 canvas 正常渲染（W≈368, H≈327），无裁切 ✓
- **残留 console 404**：仅来自验收脚本对无数据省份的候选点击，已被前端优雅处理，非缺陷。

## 四、已知限制 / 注意事项

1. **河北以外省份暂无细分数据**：当前库中所有舆情数据均在河北省。点击其他省份会展示该省市级轮廓（或联网拉取边界）并显示「该省暂无细分数据」，符合预期。如需全国下钻数据，需补齐 `Region` 表中对应省份的市/县分布数据。
2. **雄安新区边界映射**：河北省 GeoJSON 标准边界中雄安新区并入保定市，故数据中独立的「雄安新区（8 条）」在地图上无同名要素着色、保定市显示为 0——属行政区划编码与标准地图的已知差异，不影响总数与 tooltip 数据正确性。如要求雄安新区独立着色，需替换为含雄安独立要素的新版 GeoJSON。
3. **其他省份市级 GeoJSON 按需联网**：首次点击某省会从 DataV 拉取边界并前端缓存（与现有地图同源）；若运行环境禁止外网，这些省份只能显示「暂无细分数据」降级态，不影响河北省（已本地打包）。
4. **未改动契约/旧页面**：未修改 Dashboard、AppLayout、theme.css、API 契约，未引入伪造数据；旧页面回归由集成验证覆盖。
