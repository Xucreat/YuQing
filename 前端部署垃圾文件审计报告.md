## 三、部署逻辑说明（阅读 `backend/_d.py`）

- `_d.py` 仅做一件事：用 Node 把 `frontend/dist` 序列化为「路径长度+内容长度+路径+内容」的二进制流，再在 `backend/app/static` 下**按相对路径逐文件写出**（含 `index.html` 与 `assets/*`）。
- **关键特征**：
  1. **只写不删**：`fp.write_bytes(...)` 覆盖同名文件，但**从不清理目标目录中已不存在于 dist 的旧文件** → 历史 `assets.old-*`/`assets.bak*` 就是因此累积的残留。
  2. **无 fallback / 无版本切换**：不存在「按旧目录回退」的逻辑，旧目录对当前运行无任何作用。
  3. `dist` 内没有 `geo/`（它不在 vite 产物里），故 `geo/` 是独立放置的运行时静态资源，与打包无关 → 必须保留。
- 结论：所有 `assets.old-*`/`assets.bak*` 都是「上一轮构建前被重命名移走、但从未清理」的死目录；当前运行只认 `assets/`。
